# Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?
The client is named The Gaming Room and they wanted to expand on their Android game called Draw It or Lose It. They wanted me to design a multi-platform game that supported both desktop and mobile users. They also had some other requirements like rendering timed images, enforcing unique names and supporting single game instances.

# What did you do particularly well in developing this documentation?

# What about the process of working through a design document did you find helpful when developing the code?
I believe its easier to write code when you have a blueprint. I think when coding you can quickly dive down rabbit holes that both are confusing and try to tackle to many things. Utilizing the domain model made my coding more efficient.

# How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?
I think its tremendously important to realize what the client needs and build from that. Often the ask is bigger than what it may seem and so its important to quickly expand on them so you dont get blindsighted later. This could include writing alot more code or going over budget. I think it also helps to narrow the focus of the design so there is less clutter.

# How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?
I first started with what the client required and then building the domain model. This helped to identify the direction I would go in when reviewing my platform options. I utilized things like UML modeling and would continue to use this in future software designs.
